## 1. 项目定位

**明确表述：**
- 不是"自己玩得很强"的全自动 RTS AI
- 不是"LLM 写 Python 控游戏"的实验玩具
- 是 **LLM 赋能传统游戏 AI** 的系统
- 目标：Directive-Driven Multi-Expert Game Agent System

**核心动机（用户原话）：**
> LR 对于 RTS，或者 OpenRA 效果仍然不好。语料库非常难以学习，以及 AI 无法学会真正玩游戏。

> 一个简单的："包围"，其实涉及了找到 2 个好角度分兵，然后以某种顺序或者时间关系压上去打。局势改变对于操作改变都是巨大的。AI 只是在简单的模仿"包围"，把兵分成两路，一类从上方，一类从下方。（上下就是目标点加坐标偏移）这样。没有真正的思考。

**解读：** LLM 能理解语义但不能执行战术。传统 AI 能执行战术但不理解自然语言。两者结合。

## 3. 看板/Dashboard 需求

**用户原话：**
> 我同时需要一个非常不错的看板来看我这个带有时效性的任务系统，之前的看板也很乱，需要处理干净

**需求拆解：**
- 时效性任务系统的可视化看板
- 能看到任务状态流转
- 当前看板"很乱"，需要清理
- "非常不错" — 暗示高质量 UI，不是凑合用的调试面板

**已确认（2026-03-29）：**
- 看板支持多个面板 / 多模式：
  - **用户面板**：完整的最终用户界面
  - **调试面板**：开发调试用，看内部状态
- 除了看板，**日志系统也要做好**（不只是 print，要有结构化 log）

**开放问题（待确认）：**
- [ ] 是否需要看到影响力图的可视化？
- [ ] 是否需要看到战术方法的阶段进度？

## 5. 传统 RTS AI 需求

**用户原话：**
> 我要把 RTS 的 AI 真正的融入进去，而这个方案在 roadmap 里很草率，需要深度调研

**已完成的调研（rts_ai_research.md）确认：**
- 需要影响力图系统（分层：地形/视野/威胁/支援）
- 需要战术方法库（包围/夹击/撤退/扼守等有阶段状态的执行单元）
- 需要三层架构（Strategy → Tactics → Micro）
- 需要编队/分组 AI（角色分配、时间协调）
- 势场微操放在 Micro 层

**仍需细化：**
- [ ] 具体哪些战术方法是第一优先？
- [ ] 影响力图的精度要求？（tile 级？区域级？）
- [ ] 对手 AI（EnemyAgent）是否也使用新架构？还是独立系统？
