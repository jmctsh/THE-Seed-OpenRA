# yu Knowledge Base

- Deliverables explicitly assigned by Wang go under `docs/wang/` (or `docs/wang/archive/` for archived investigation artifacts); `docs/yu/` is reserved for yu's own agent state and yu-only files.
- Base `TaskAgent` now has native `push_player_response()` intake that re-injects `PlayerResponse` as a normal wake event; WS / Adjutant reply flow can route through `Kernel.submit_player_response()` directly.
- Test suites that replace the shared runtime surface with custom mocks can hide integration blockers; before clearing a milestone, check the mock API shape against the real `openra_api` / `Kernel` / `WorldModel` contracts.
- `openra_api.GameAPI` is stateless per request, so OpenRA process restarts do not require recreating the API client; runtime reattachment only needs `wait_for_api()` plus `WorldModel.reset_snapshot()` and a forced refresh.
- In the live runtime, `GameLoop` currently runs on the same asyncio loop as WS/Adjutant/LLM coroutines; because `WorldModel.refresh()` and job ticks call synchronous `GameAPI` sockets, slow game polling can starve `asyncio.wait_for()` and make LLM calls appear to hang even when the provider itself is healthy.
- In `OpenCodeAlert`, GameAPI request spam can come from `CopilotCommandServer.AcceptAsync()` connection logs rather than the JSON payload dumps; the payload dumps are already behind `CopilotDebug`, but per-connection accept/stop-loop logs must also be gated if the Python side opens one short-lived socket per request.
- `GameAPI` long-connection reuse requires protocol framing as well as socket reuse: the old `send once -> wait for EOF/timeout` behavior works only because the server closed the socket after each response. To support persistent connections safely, both client and `CopilotCommandServer` must use a real message boundary (newline framing here), with fallback parsing for one-shot legacy packets.
- Kernel actor allocation must treat `building` / `static` actors as opt-in resources. Soft needs like `{"owner": "self"}` should only claim mobile actors by default; jobs that truly want structures must say so explicitly in their `ResourceNeed` predicates.
- For emergency auto-responses such as `BASE_UNDER_ATTACK`, creating a managed task and waiting for TaskAgent/LLM is too slow for first reaction. The Kernel should directly start or retarget a concrete expert job, and let the task/LLM layer handle follow-up reasoning afterward.
- User-initiated synchronous feedback (`command`, `reply`, `query`) should go through the chat/adjutant response channel, not `player_notification`. `player_notification` is suitable for async system/task events; routing direct command acks there makes the UI feel like the command was swallowed.
- In the Vue chat frontend, long-lived `localStorage` history plus top-level WS listener registration can create “stale cached dialogue” and duplicate message replay. Chat history should be session-scoped, and WS subscriptions should unregister on component unmount.
